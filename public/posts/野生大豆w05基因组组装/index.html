野生大豆W05基因组组装

PacBio CLR + Illumina 的野生大豆基因组组装实操教程，含完整代码与参数解释
2026-02-07 · 5 min · tdx

本文给出一套可直接复用的实操流程，适用于 PacBio CLR + Illumina 数据组合，目标是得到高质量 contig，并可继续扩展到染色体级（Bionano + Hi-C）。

文章结构：

  1. 先给一份可运行主脚本（从质控到评估）
  2. 再逐段解释每一步代码的作用、输入输出和关键参数
  3. 最后给染色体级挂载的可选步骤

一、数据与策略#

推荐数据覆盖度（以约 1.0 Gb 植物基因组为例）：

  • PacBio CLR：>= 60x，建议 80x+
  • Illumina PE150：>= 50x

核心策略：

  • 长读长负责连续性：de novo 组装
  • 去冗余降低假重复：purge_dups
  • 长读长纠错 + 短读长抛光提升碱基准确率：Racon -> Pilon
  • 多指标评估：QUAST + BUSCO + Merqury

二、目录规范#

project/
  raw_data/
    pacbio_clr.fastq.gz
    illumina_R1.fastq.gz
    illumina_R2.fastq.gz

三、主流程一键脚本（可直接改路径运行）#

将以下脚本保存为 run_w05_clr_assembly.sh。

#!/usr/bin/env bash
set -euo pipefail

########################################
# 0) 用户参数区（按实际情况修改）
########################################
THREADS=48
GENOME_SIZE="1.05g"
WORKDIR="/data/w05_assembly"

PB_READS="/data/raw/pacbio_clr.fastq.gz"
ILL_R1="/data/raw/illumina_R1.fastq.gz"
ILL_R2="/data/raw/illumina_R2.fastq.gz"

# 如果系统没有 pilon 命令，请设置 PILON_JAR
# export PILON_JAR=/path/to/pilon.jar
PILON_JAVA_MEM="200G"

########################################
# 1) 工具检查
########################################
REQUIRED_TOOLS=(
  fastp jellyfish flye minimap2 samtools bwa racon
  pbcstat calcuts split_fa purge_dups get_seqs
  quast.py busco meryl merqury.sh seqkit
)

for t in "${REQUIRED_TOOLS[@]}"; do
  command -v "$t" >/dev/null 2>&1 || { echo "[ERROR] Missing: $t"; exit 1; }
done

if command -v pilon >/dev/null 2>&1; then
  PILON_CMD="pilon"
elif [[ -n "${PILON_JAR:-}" ]] && [[ -f "${PILON_JAR}" ]]; then
  PILON_CMD="java -Xmx${PILON_JAVA_MEM} -jar ${PILON_JAR}"
else
  echo "[ERROR] Pilon not found. Install pilon or set PILON_JAR"
  exit 1
fi

if command -v genomescope2 >/dev/null 2>&1; then
  HAS_GENOMESCOPE=1
else
  HAS_GENOMESCOPE=0
fi

########################################
# 2) 建目录
########################################
mkdir -p "${WORKDIR}"/{raw_data,00_qc,01_kmer,02_asm,03_purge,04_polish,05_eval,logs}
cd "${WORKDIR}"

########################################
# 3) CLR基础质控（统计，不做过度剪切）
########################################
seqkit stats -a "${PB_READS}" > 00_qc/pacbio_seqkit_stats.txt

########################################
# 4) Illumina质控
########################################
fastp \
  -i "${ILL_R1}" -I "${ILL_R2}" \
  -o 00_qc/clean_R1.fq.gz -O 00_qc/clean_R2.fq.gz \
  -w "${THREADS}" \
  -h 00_qc/fastp.html \
  -j 00_qc/fastp.json \
  2>&1 | tee logs/01_fastp.log

########################################
# 5) k-mer估计
########################################
jellyfish count \
  -C -m 21 -s 20G -t "${THREADS}" \
  <(zcat 00_qc/clean_R1.fq.gz 00_qc/clean_R2.fq.gz) \
  -o 01_kmer/mer21.jf \
  2>&1 | tee logs/02_jellyfish_count.log

jellyfish histo -t "${THREADS}" 01_kmer/mer21.jf > 01_kmer/mer21.histo

if [[ "${HAS_GENOMESCOPE}" -eq 1 ]]; then
  genomescope2 -i 01_kmer/mer21.histo -k 21 -o 01_kmer/genomescope_k21 \
    2>&1 | tee logs/03_genomescope2.log
fi

########################################
# 6) Flye组装（PacBio CLR）
########################################
flye \
  --pacbio-raw "${PB_READS}" \
  --genome-size "${GENOME_SIZE}" \
  --threads "${THREADS}" \
  --out-dir 02_asm/flye \
  2>&1 | tee logs/04_flye.log

########################################
# 7) purge_dups去冗余
########################################
cp 02_asm/flye/assembly.fasta 03_purge/assembly.fasta
cd 03_purge

minimap2 -x map-pb -t "${THREADS}" assembly.fasta "${PB_READS}" | \
  samtools sort -@ "${THREADS}" -o pb2asm.bam
samtools index pb2asm.bam

pbcstat pb2asm.bam
calcuts PB.stat > cutoffs

split_fa assembly.fasta > assembly.split.fa
minimap2 -xasm5 -DP -t "${THREADS}" assembly.split.fa assembly.split.fa | gzip -c > self.paf.gz

purge_dups -2 -T cutoffs -c PB.base.cov self.paf.gz > dups.bed
get_seqs -e dups.bed assembly.fasta
# 关键输出: purged.fa

cd "${WORKDIR}"

########################################
# 8) 长读长抛光（Racon 2轮）
########################################
cp 03_purge/purged.fa 04_polish/genome.r0.fa

for i in 1 2; do
  prev=$((i-1))
  minimap2 -x map-pb -t "${THREADS}" "04_polish/genome.r${prev}.fa" "${PB_READS}" > "04_polish/pb.r${i}.paf"
  racon -t "${THREADS}" "${PB_READS}" "04_polish/pb.r${i}.paf" "04_polish/genome.r${prev}.fa" > "04_polish/genome.r${i}.fa"
done

########################################
# 9) 短读长抛光（Pilon 2轮）
########################################
cp 04_polish/genome.r2.fa 04_polish/genome.p0.fa

for i in 1 2; do
  prev=$((i-1))
  bwa index "04_polish/genome.p${prev}.fa"

  bwa mem -t "${THREADS}" "04_polish/genome.p${prev}.fa" 00_qc/clean_R1.fq.gz 00_qc/clean_R2.fq.gz | \
    samtools sort -@ "${THREADS}" -o "04_polish/ilmn.p${i}.bam"
  samtools index "04_polish/ilmn.p${i}.bam"

  ${PILON_CMD} \
    --genome "04_polish/genome.p${prev}.fa" \
    --frags "04_polish/ilmn.p${i}.bam" \
    --output "genome.p${i}" \
    --outdir 04_polish \
    --threads "${THREADS}" \
    --changes --vcf \
    2>&1 | tee "logs/05_pilon_round${i}.log"
done

########################################
# 10) 评估
########################################
FINAL_FA="04_polish/genome.p2.fasta"

quast.py -t "${THREADS}" -o 05_eval/quast "${FINAL_FA}" \
  2>&1 | tee logs/06_quast.log

busco \
  -i "${FINAL_FA}" \
  -l embryophyta_odb10 \
  -m genome \
  -c "${THREADS}" \
  -o busco_w05 \
  --out_path 05_eval \
  2>&1 | tee logs/07_busco.log

meryl count k=21 output 05_eval/read_db.meryl 00_qc/clean_R1.fq.gz 00_qc/clean_R2.fq.gz
merqury.sh 05_eval/read_db.meryl "${FINAL_FA}" 05_eval/merqury \
  2>&1 | tee logs/08_merqury.log

echo "[DONE] Final assembly: ${WORKDIR}/${FINAL_FA}"

运行：

chmod +x run_w05_clr_assembly.sh
./run_w05_clr_assembly.sh

四、代码解释（按步骤）#

Step 1 工具检查#

作用：确保脚本中每个命令都存在，避免跑到中途才报错。

关键点：

  • command -v 用于检查命令是否可调用
  • Pilon 兼容两种形式：pilon 命令或 java -jar pilon.jar

Step 2 建目录#

作用：统一输入输出路径，保证流程可追溯。

目录职责：

  • 00_qc：质控输出和报告
  • 01_kmer：k-mer 计数与估计
  • 02_asm：初组装结果
  • 03_purge：去冗余结果
  • 04_polish：抛光结果
  • 05_eval：评估报告

Step 3 CLR 质控（统计优先）#

作用：检查 PacBio CLR 的长度分布和总产量，不建议像二代那样强剪切。

为什么只统计：

  • CLR 错误率高是技术特性，后续通过抛光修正
  • 过度按质量值过滤会降低覆盖，影响组装完整性

Step 4 Illumina 质控#

作用：去接头、去低质量、保留高可信短读长用于后续抛光。

关键参数：

  • -w：线程数
  • -h/-j：生成报告，便于核查保留率和质量分布

Step 5 k-mer 估计#

作用：估算基因组大小、杂合度、重复比例，帮助判断组装结果是否异常。

关键参数：

  • -m 21：常用 k 值，速度和分辨率平衡
  • -C：合并互补链计数

Step 6 Flye 组装#

作用：使用 CLR 构建初始 contig。

关键参数：

  • --pacbio-raw：指定 CLR 原始长读长模式
  • --genome-size：建议使用 k-mer 估计结果

输出：02_asm/flye/assembly.fasta

Step 7 purge_dups 去冗余#

作用：移除等位冗余和单倍型重复，避免组装体积虚高。

流程逻辑：

  1. CLR 回贴组装得到深度信息
  2. pbcstat + calcuts 推断深度阈值
  3. 自比对识别重复片段
  4. 输出 purged.fa 作为主组装

Step 8 Racon 长读长抛光#

作用：优先修复长片段层面的 indel 错误。

为什么 2 轮：

  • 第 1 轮纠正明显错误
  • 第 2 轮进一步收敛
  • 通常 3 轮后收益变小

Step 9 Pilon 短读长抛光#

作用：利用 Illumina 的高准确率修复 SNP 和小 indel。

为什么在 Racon 后执行：

  • 先把结构层面的长读长误差降下来
  • 再用短读长做碱基层面精修，效果更稳定

Step 10 评估#

作用：从连续性、完整性、准确性三方面确认结果质量。

  • QUAST：N50、总长度、片段数
  • BUSCO：保守基因完整度
  • Merqury：k-mer 一致性与 QV

五、关键结果判读建议#

常见合理区间（植物二倍体，约 1.0 Gb）：

  • 组装总长接近 k-mer 估计大小（误差通常在 5% 内）
  • BUSCO 完整度：>= 95%
  • BUSCO duplicated 不宜异常偏高（高则考虑加严去冗余）
  • Merqury QV：越高越好，通常 QV > 35 说明已较可靠

六、可选扩展：做染色体级（Bionano + Hi-C）#

当你已有 purged + polished 的高质量 contig 后，可继续：

  1. Bionano Solve 做 hybrid scaffold
  2. Juicer 构建 Hi-C contact map
  3. 3D-DNA 做排序定向
  4. Juicebox 手动修正后导出 final assembly

最小示意命令：

# 1) Juicer (示意)
juicer.sh -g Gsoja -s MboI -z contigs.fa -p MboI_sites.txt -y MboI_sites.txt -D /path/to/juicer

# 2) 3D-DNA (示意)
run-asm-pipeline.sh -r 2 contigs.fa aligned/merged_nodups.txt

说明：

  • 这一部分强依赖实验设计和平台版本，不同实验室参数差异较大
  • 建议将 CLR+Illumina 流程先稳定跑通，再进入染色体级挂载

七、常见报错与处理#

  1. flye 失败或超慢
  • 检查 CLR 实际覆盖度是否足够
  • 降低并发线程，观察内存是否打满
  1. purge_dups 后组装长度异常小
  • 检查 calcuts 阈值是否合理
  • 先保守参数运行，再逐步加强去冗余
  1. Pilon 内存溢出
  • 使用 PILON_JAR 时提高 -Xmx
  • 或按染色体/大片段分块抛光
  1. BUSCO 低于预期
  • 回查污染和去冗余是否过强
  • 回查是否错误使用了非目标物种数据库

八、结论#

对于野生大豆这类基因组，PacBio CLR + Illumina 完全可以完成高质量组装。推荐的稳定顺序是：

CLR组装 -> 去冗余 -> 长读长抛光 -> 短读长抛光 -> 质量评估 -> (可选)染色体级挂载
